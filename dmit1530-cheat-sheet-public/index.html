Relative units

Relative units define lengths and sizes in relation to other elements offering flexibility and responsiveness in web design, there are 4 main relative units that we will need to know: em, rem, vw, and vh

em

em is relative to the font-size of the element itself, it's relative to the parent elements font-size, the h3 of this div is set to a font-size of 1.5em meaning it's 1.5 times the original h3 font size

rem

rem is "relative to the root element" meaning it's set to the default font-size of the html element, by default it's 16px so if it's not specified 1rem = 16px, we will be using rem a lot in this course. The font-size of the h3 of this div is set to 1.5rem meaning it's 1.5 times the html font size

vh

vh is "viewport height" which refers to the visible area of a web page in a percentage-based calculation based off the current height of the browser, if the viewport is 1000 pixels high, 1vh would be 10 pixels, and 100vh would be 1000 pixels, filling the entire viewport height

vw

vw is "viewport width" which refers to the visible area of a web bage in a percentage-based calculation based off the current width of the browser, if the viewport is 1200 pixels wide, 1vw would be 12 pixels, and 10vw would be 120 pixels.

Fluid design

Fluid design uses the calc function to determine width of objects in a web page that changes in width, hence "fluid" design, first you put the entirity, this utilizes the following formula:

width: calc((X - Y) / Z)

Where:

- X is the percentage of the max-width you want the design to take up
- Y is the gap count in rem
- Z is the size of the object in relation to other objects in the same row


This section itself just has it's width as 100% since there's nothing else in this row, also note that the entire thing is flexed
